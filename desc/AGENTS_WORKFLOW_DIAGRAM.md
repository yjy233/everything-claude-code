# Agents 工作流程图解

## 简单理解

```
┌─────────────┐      ┌─────────────┐      ┌─────────────────┐
│  Markdown   │  +   │    AI       │  =   │  专业智能体      │
│  文件       │      │  (Claude)   │      │  (角色模式)      │
└─────────────┘      └─────────────┘      └─────────────────┘

  (剧本/剧本)           (演员)              (角色扮演)
```

## 不同触发方式对比

### 方式 1: 命令触发 (显式)

```
用户: /plan "添加支付功能"
       │
       ▼
  commands/plan.md
       │
       ▼
  "Delegate to planner"
       │
       ▼
  agents/planner.md ◄── 加载并执行
```

### 方式 2: 自动委托 (隐式)

```
用户: 帮我添加一个支付功能
       │
       ▼
  Claude 分析: "这是复杂功能请求"
       │
       ▼
  AGENTS.md: "Complex features → planner"
       │
       ▼
  agents/planner.md ◄── 自动加载并执行
```

### 方式 3: Hook 触发 (事件驱动)

```
用户写完代码 ──► 保存文件
                    │
                    ▼
              hooks/post-save
                    │
                    ▼
              "Consider code-reviewer"
                    │
                    ▼
              agents/code-reviewer.md ◄── 触发审查
```

## 总结

| 组件 | 类比 | 作用 |
|------|------|------|
| `CLAUDE.md` | 公司入职手册 | 告诉 Claude 项目背景 |
| `AGENTS.md` | 组织架构图 | 告诉 Claude 什么时候找谁 |
| `agents/*.md` | 职位说明书 | 告诉 Claude 具体怎么做 |
| `commands/*.md` | 快捷拨号 | 快速触发特定 Agent |
| `hooks/` | 自动化规则 | 特定事件自动触发 |

**核心原理**: 
> 这些 Markdown 文件是**结构化的系统提示模板**。Claude Code 读取它们，将内容注入到对话中，使 Claude 表现出特定专业角色的行为模式。
