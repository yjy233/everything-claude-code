## 核心组件详解

### 1. 智能体 (Agents)

**设计理念**: 专业分工，按需委托

```
用户请求
    │
    ▼
┌──────────────┐
│ 智能体路由器 │ ← 根据请求类型选择智能体
└──────┬───────┘
       │
       ├──► planner (规划类)
       ├──► tdd-guide (测试类)
       ├──► code-reviewer (审查类)
       ├──► security-reviewer (安全类)
       └──► ... (其他专业智能体)
```

**智能体类型:**
- **流程类**: planner, architect, tdd-guide
- **审查类**: code-reviewer, security-reviewer
- **语言类**: typescript-reviewer, python-reviewer, go-reviewer...
- **专项类**: database-reviewer, e2e-runner, build-error-resolver

### 2. 技能 (Skills)

**设计理念**: 可复用的领域知识

```
skills/
├── [workflow]/
│   └── SKILL.md    ← 包含完整工作流定义
```

**技能结构:**
```markdown
# 技能名称

## 何时使用
- 触发条件
- 适用场景

## 工作流步骤
1. 步骤一
2. 步骤二

## 示例
- 代码示例
- 最佳实践

## 注意事项
- 常见陷阱
- 优化建议
```

### 3. 命令 (Commands)

**设计理念**: 快速执行，标准化流程

```
命令执行流程:
用户输入 /command → 解析参数 → 加载上下文 → 委托智能体 → 执行 → 返回结果
```

**命令类型:**
- **开发命令**: /plan, /tdd, /code-review, /build-fix
- **多智能体**: /multi-plan, /multi-execute, /orchestrate
- **学习命令**: /learn, /evolve, /instinct-status
- **管理命令**: /sessions, /pm2, /setup-pm

### 4. 规则 (Rules)

**设计理念**: 始终遵循的编码准则

```
规则层级:
├── common/          # 所有项目通用
│   ├── coding-style.md
│   ├── testing.md
│   └── security.md
├── typescript/      # TypeScript 专用
├── python/          # Python 专用
└── ...             # 其他语言
```

**核心规则:**
1. **不可变性** - 永不修改现有对象
2. **小文件** - 200-400 行典型
3. **TDD** - 80%+ 覆盖率
4. **错误处理** - 每个层级都要处理

### 5. 钩子 (Hooks)

**设计理念**: 事件驱动的自动化

```
钩子生命周期:
SessionStart → PreToolUse → [Tool执行] → PostToolUse → ... → Stop
     │            │                             │            │
     ▼            ▼                             ▼            ▼
 加载上下文    前置检查                      结果处理    保存状态
```

**钩子类型:**
- `SessionStart` - 会话开始时加载上下文
- `PreToolUse` - 工具执行前验证
- `PostToolUse` - 工具执行后处理
- `Stop` - 会话结束时保存状态

## 扩展机制

### 1. 添加新智能体

```
1. 创建 agents/new-agent.md
2. 定义目的、触发条件、工作流程
3. 更新 AGENTS.md 注册
```

### 2. 添加新技能

```
1. 创建 skills/new-skill/SKILL.md
2. 遵循技能模板格式
3. 添加到技能目录
```

### 3. 添加新命令

```
1. 创建 commands/new-command.md
2. 定义用法、参数、示例
3. 更新相关文档
```
